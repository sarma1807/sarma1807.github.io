Phase 0: Prerequisites & Hardware Check

Before implementing, ensure your server supports hardware acceleration for encryption.

LINUX OS COMMAND 
# Check if the CPU supports AES instructions
grep -o aes /proc/cpuinfo | head -1
ORACLE SQL COMMANDS 
-- 1. Confirm 19c and CDB architecture
SELECT version, cdb FROM v$database ;

-- 2. Verify all PDBs are in READ WRITE mode (Required for key settings)
SELECT name, open_mode FROM v$pdbs ;

-- 3. Check for legacy wallet settings (Should be empty for WALLET_ROOT)
SHOW PARAMETER wallet ;

Phase 1: Infrastructure & Keystore

# Create OS directory
mkdir -p  /u01/app/oracle/admin/CDB/wallet 

-- Define OS directory and configuration type
ALTER SYSTEM SET WALLET_ROOT = '/u01/app/oracle/admin/CDB/wallet' SCOPE=SPFILE ;

-- IMPORTANT: Restart database here --
SHUTDOWN IMMEDIATE ;
STARTUP ;

ALTER SYSTEM SET TDE_CONFIGURATION="KEYSTORE_CONFIGURATION=FILE" SCOPE=BOTH ;

-- Create the password-protected keystore
ADMINISTER KEY MANAGEMENT CREATE KEYSTORE IDENTIFIED BY "Pass123" ;
-- Create the Auto-Login file (cwallet.sso)
ADMINISTER KEY MANAGEMENT CREATE AUTO_LOGIN KEYSTORE FROM KEYSTORE IDENTIFIED BY "Pass123" ;

Phase 2: Master Key Activation

-- Open the keystore (if not auto-login)
ADMINISTER KEY MANAGEMENT SET KEYSTORE OPEN IDENTIFIED BY "Pass123" ;

-- Set Master Key for CDB Root
ADMINISTER KEY MANAGEMENT SET KEY IDENTIFIED BY "Pass123" WITH BACKUP CONTAINER=CURRENT ;
-- Set Master Key for ALL PDBs
ADMINISTER KEY MANAGEMENT SET KEY IDENTIFIED BY "Pass123" WITH BACKUP CONTAINER=ALL ;

Encrypt Data

-- Create new encrypted tablespace
CREATE TABLESPACE  secure_ts
DATAFILE '/path/to/datafile.dbf' SIZE 100M
ENCRYPTION USING 'AES256' DEFAULT STORAGE (ENCRYPT) ;

-- Encrypt existing tablespace
ALTER TABLESPACE  users ENCRYPTION OFFLINE ENCRYPT ;

Rotating the Master Encryption Key

-- Rotate key for the entire CDB and all PDBs
ADMINISTER KEY MANAGEMENT SET KEY IDENTIFIED BY "YourPassword123"
WITH BACKUP USING 'backup_before_rotation'
CONTAINER = ALL ;

-- Rotate key for a specific PDB only (must be connected to that PDB)
ALTER SESSION SET CONTAINER = PDB1 ;
ADMINISTER KEY MANAGEMENT SET KEY IDENTIFIED BY "YourPassword123"
WITH BACKUP CONTAINER = CURRENT ;

Phase 3: Algorithm Performance Matrix

If the hardware check in Phase 0 returned aes, AES256 is the recommended standard.

Algorithm Security CPU Overhead (with AES-NI) Best Use Case
AES128 High Minimal (<1%) High Performance Apps
AES192 Very High Low (~2%) Standard Security
AES256 Maximum Low (~3%) Compliance (PCI, HIPAA)

Phase 4: Unplug/Plug Encrypted PDB

Unplug encrypted PDB from Source CDB

-- SOURCE: Export PDB keys for transport
ADMINISTER KEY MANAGEMENT EXPORT KEYS WITH SECRET "temp_secret"
TO '/tmp/pdb_keys.exp' IDENTIFIED BY "WalletPass" CONTAINER=pdb1 ;

-- Unplug the PDB as usual
ALTER PLUGGABLE DATABASE pdb1 CLOSE IMMEDIATE ;
ALTER PLUGGABLE DATABASE pdb1 UNPLUG INTO '/tmp/pdb1.xml' ;

Plug encrypted PDB into destination CDB

-- Create PDB from XML
CREATE PLUGGABLE DATABASE pdb1 USING '/tmp/pdb1.xml' NOCOPY ;

-- Import the keys
ADMINISTER KEY MANAGEMENT IMPORT KEYS WITH SECRET "temp_secret"
FROM '/tmp/pdb_keys.exp' IDENTIFIED BY "DestWalletPass" CONTAINER=pdb1 ;

-- Open PDB
ALTER PLUGGABLE DATABASE pdb1 OPEN ;

Phase 5: Final Verification

-- show parameters
SHOW PARAMETER  wallet_root ;
SHOW PARAMETER  tde_configuration ;

-- Verify wallet is OPEN and AUTOLOGIN
SELECT wrl_type wallet_type, wrl_parameter wallet_params, status FROM v$encryption_wallet ;

-- View All Keys
SELECT con_id, key_id, creation_time FROM v$encryption_keys ;

-- Check for encrypted tablespaces
SELECT tablespace_name, encrypted FROM dba_tablespaces ;
CRITICAL SAFETY: Never delete your wallet backups. Losing ewallet.p12 or cwallet.sso leads to absolute data loss. Always backup these files.
